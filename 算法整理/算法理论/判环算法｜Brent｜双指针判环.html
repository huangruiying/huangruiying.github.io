高效判环，但无法找到环的入口 ​

与Floyd比较，Brent缩短判环耗时38%。

算法原理 ​

同样会使用快和慢两个指针，也是根据两个指针是否会相遇进行判环。但是快慢指针走法不同

慢指针走n步，快指针每一次会向前 2n 步。快指针走完后，慢指针传送到快指针的位置

text
n=1 慢指针起始 0，走到 1 ，快指针起始 0，走 2 步到4，每步都与慢指针比较是否相遇
n=2 慢指针起始 2，走到 3 ，快指针起始 2，走 4 步到6，每步都与慢指针比较是否相遇
n=2 慢指针起始 6，走到 7 ，快指针起始 6，走 8 步到14，每步都与慢指针比较是否相遇

代码示例 ​

java
 public boolean hasCycle(ListNode head) {
	ListNode slow = head;
	ListNode fast = head;

	// 当前回合快指针已走的步数
	int stepsMovedByFast = 0;
	// 当前回合快指针最多能走的步数
	int stepLimitForFast = 2;

	while (fast != null && fast.next != null) {
		fast = fast.next;
		stepsMovedByFast++;

		// 如果快指针往前走的时候与慢指针相会
		// 那就说明链表中存在环
		if (fast == slow) {
			return true;
		}

		// 快指针走完了能走的步数，即回合结束
		if (stepsMovedByFast == stepLimitForFast) {
			// 快指针在下一回合能走的步数翻倍
			stepLimitForFast *= 2;
			// 清零计步器
			stepsMovedByFast = 0;

			// 慢指针传送到快指针所在的位置
			slow = fast;
		}
	}

	return false;
}
