数据对比流程

  1. 通过代码，捞取DB中固定数据，指定对账双方的对账关系和对账字段，进行比对
    1. 缺点：代码需要大量定制，不通用，其他业务线接入对账的成本较高。
  2. 塑造对账模板（SQL + 别名 + 条件）、对账配置。
    1. 自定义数据模型；通过配置SQL的方式，定义多个数据模板，并有数据标记，用于区分不同业务角度的数据(业务支付单、三方通道支付单等)。
    2. 自定义对账配置；通过选择数据模型，生成对账配置，供对账程序使用。
    3. 高性能对账程序；开发一套通用的对账程序，读取对账配置并对账。
      1. 根据配置指定执行频次、数据捞取范围等数据模型需要的参数，并拿到数据模型进行数据的获取与对比。
  3. 使用外部能力，如数据灌入Hive，利用HiveSQL，将数据导出为 核对成功、存疑数据 表。
    1. 核对成功：通过SQL，对本端的单边账，结果写入表
    2. 存疑数据：对于两端数据非完全一致的，标记数据存疑(可能为 )
      1. 通过SQL，对本端的单边账
      2. 通过SQL，对渠道/通道端的单边账
      3. 使用 SQL union 将 1 2 结果合并，并写入表
    3. 二次存疑数据处理：针对单边对账时无法区分的差异，进行双边核对
    4. 数据汇总： DP 平台 Sprak 任务进行处理
